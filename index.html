Wrapper Era · Phase 0

Audit-grade output for the Wrapper Era.

Provenance is the MCP server that surfaces SOX evidence chains, tickmark conventions, workpaper section scaffolds, management-assertion inputs, and a defensibility validator — all consumed by a foundation-model agent like Claude Desktop.

Companion to Context: where Context answers "where are we", Provenance answers "prove it." The Controller never opens the Optro UI — they ask Claude and get back something a Big 4 reviewer will accept, with every claim cited back to an optro_record_id.

THE FRAMING

In the Wrapper Era, the foundation model is the workpaper-drafting interface. A pre-IPO Controller doesn't open Optro — they ask Claude Desktop questions like "draft the management assertion section for our S-1" or "show me the evidence chain for control RC-047."

Provenance is the evidence-of-truth backend that makes the agent's output defensible. Structured responses with explicit citations, tickmark vocabulary, and REQUIRES HUMAN REVIEW markers wherever judgment was applied — so the model can't fabricate evidence and the customer can always trace a claim back to source.

Audit-grade by construction

— five primitives keep the foundation model honest and the human in the loop.

Drafting workpapers with an LLM is the easy part. Making the output defensible — traceable, attested, and reviewed — is what separates a Wrapper Era prototype from something a pre-IPO Controller can hand to Deloitte. Provenance enforces the five primitives below at the API boundary, so the agent cannot ship audit output without them.

01 · WATERMARK
Provenance signature on every response
Every tool response is wrapped in a tamper-evident envelope: schema_version, source_system, generated_at, optro_record_id. If a fact in the agent's output doesn't trace to a record_id, it didn't come from Optro — and a reviewer can tell at a glance.
"provenance": {
  "source_system": "optro-provenance-preipo",
  "schema_version": "0.1.0",
  "optro_record_id": "rec_chain_rc_047"
}
02 · TRACEABILITY
Every claim cites an optro_record_id
When the agent says "4 orphaned production accounts were identified", that fact is bound to a specific Optro record. The agent renders the citation pill inline so a Big 4 reviewer can drill from prose back to evidence — no hallucinated numbers, no fabricated artifact ids.
rec_art_it001_l1  rec_chain_fr_002
03 · SAFETY BOUNDARY
REQUIRES HUMAN REVIEW markers stay verbatim
Wherever judgment was applied — severity classification, scope caveats, missing-evidence remediation — Provenance returns an explicit review marker that the agent must preserve. The model literally cannot draft past these points without flagging them to the Controller.
slot["step_4"]: "⚠ REQUIRES HUMAN REVIEW:
  step 4 has no evidence artifact captured.
  Document the outcome before workpaper sign-off."
04 · READINESS GAUGE
Defensibility score (0 – 100)
A numerical readout of chain completeness with severity-banded issues. Controllers see immediately whether a workpaper would survive a Big 4 review — and what specifically would fail it. Below 80 = remediation needed; below 50 = blocking.
075100
05 · WORKFLOW GATE
Required human sign-offs before filing
Three signatures required on every workpaper: Preparer, Independent Reviewer, External Auditor. The agent renders the ribbon with status pills (SIGNED / AWAITING / BLOCKED) and cannot mark the workpaper complete until all three are captured.
✓ PREPARER
⚠ REVIEWER
✓ AUDITOR
Why this matters. Shneiderman's HCAI framework calls this the "Reliable, Safe, Trustworthy" zone — high automation paired with high human control. The primitives above are how Provenance gets there: visibility of state (provenance strip), recognition over recall (inline citation pills), error prevention (REQUIRES HUMAN REVIEW gates), match to the real world (Deloitte tickmarks + workpaper conventions), and explicit human checkpoints (sign-off ribbon). The model assists. The Controller decides. The Auditor verifies. None of those handoffs happen by accident.

Full strategic analysis
Scenario story
By 2027 Anthropic and OpenAI ship SOX-aware agents directly with audit reasoning and compliance memory. The agents will draft management assertions and workpapers — but only as well as the structured evidence they can call. Vertical players who survive are the ones whose data and output discipline are wired in as MCP first, before the model vendors pick a default backend.
What this prototype proves
  • Installable, runnable MCP server — npx tsx server.ts boots cleanly on stdio.
  • 5 tools registered: get_evidence_chain, get_tickmark_legend, draft_workpaper_section, list_management_assertion_inputs, validate_audit_trail.
  • Provenance envelope on every response — schema_version, optro_record_id, generated_at — so the agent can't claim what Optro didn't return.
  • One engineer can ship this in <1 day from scenario plan to functional artifact.
What it does not address
  • Real Optro tenant authentication — Phase 0 trusts local client.
  • HTTP transport for remote-tenant access.
  • ChatGPT plugin / OpenAI compatibility.
  • Multi-tenant routing and per-customer evidence stores.
Trigger conditions
Anthropic ships a SOX-aware agent AND 2+ funded wrappers raise $20M+ rounds aimed at audit-draft workflows → double down on data-first + output-discipline architecture; ship Provenance immediately alongside Context.
Cost · win condition
2 engineers · 1 quarter for the production version. Win = when Anthropic/OpenAI ship a SOX agent, Optro's MCP is the preferred backend because the audit-grade output layer is already wired in and nothing else returns citations Big 4 reviewers will accept.
