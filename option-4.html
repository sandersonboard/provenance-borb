Setup walkthrough

Wire Provenance into Claude Desktop.

Three steps. Two minutes. Then ask Claude to draft a workpaper.

Install

1
Clone and install
Pulls the MCP server and the @modelcontextprotocol/sdk dependency.
# clone the repo
git clone git@github.com:sandersonboard/provenance-borb.git
cd provenance-borb

# install dependencies
npm install

# verify every handler returns clean shapes (40+ assertions)
npm run smoke
2
Add Provenance to your claude_desktop_config.json
Replace /absolute/path/to/provenance-borb with where you cloned the repo. Config locations:
· macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
· Windows: %APPDATA%\Claude\claude_desktop_config.json
{
  "mcpServers": {
    "optro-provenance-preipo": {
      "command": "npx",
      "args": ["tsx", "/absolute/path/to/provenance-borb/server.ts"]
    }
  }
}
Provenance pairs naturally with Context. Add both mcpServers entries and Claude will pick the right tool depending on whether the user is asking "where are we" (Context) or "prove it" (Provenance).
3
Restart Claude Desktop and ask a question
Quit and re-open Claude Desktop. The server registers as optro-provenance-preipo and you'll see "5 tools available" in the MCP indicator. Then try one of:
"Draft the management assertion section for Helios's S-1 based on current testing status."
"Show me the evidence chain for control RC-047 in workpaper form."
"Run a defensibility check across our three open significant deficiencies."

The 5 tools

get_evidence_chain(control_id)
Full audit-grade chain for one control: definition, owner, frequency, test procedures, evidence artifacts (with ids and types — screenshot, query_output, attestation, sample, reconciliation, log_export), reviewer attestation, overrides applied, final disposition, defensibility_score, issues.
→ control_id: string (e.g., "RC-047", "FR-007", "IT-001")
get_tickmark_legend(control_id)
Deloitte-standard tickmark conventions. Each symbol → definition + procedure. Response distinguishes symbols actually used in this control's workpapers from the full legend, so the agent annotates procedure language correctly.
→ control_id: string
draft_workpaper_section(control_id, section)
Structured slots — not prose — for one workpaper section. Header, slots[], citations, and explicit REQUIRES HUMAN REVIEW markers wherever judgment was applied. The agent renders prose downstream using slot values verbatim; structure prevents hallucinated evidence.
→ control_id: string
→ section: "narrative" | "testing" | "exception" | "remediation"
list_management_assertion_inputs()
Inputs for the S-1 management assertion: scope of program (consolidation + exclusions), must-have universe counts, testing completeness, open findings by severity (material weakness / significant deficiency / deficiency), remediation status, disclosure recommendation with proposed caveat language.
→ no parameters
validate_audit_trail(control_id)
Defensibility check on one control's chain. Returns defensibility_score (0-100) and issues[] — each with severity (blocking/high/medium/low), category (missing_evidence / missing_reviewer / missing_override_reason / missed_cycle / stale_attestation), and remediation_suggestion. Call on every must-have control before signing the management assertion.
→ control_id: string
PROVENANCE SIGNATURE
Every tool response is wrapped in a provenance envelope: generated_at, source_system, schema_version, optro_record_id. Plus every artifact, procedure, reviewer, and override carries its own optro_record_id. When Claude says "4 orphaned accounts were identified", the user can trace it back to rec_art_it001_l1 in Optro.
